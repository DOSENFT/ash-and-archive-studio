## What this spec is (and is not)
SPEC-002 sealed the *function*: `compose(stance, gameState, entryGraph, riteSet, budgets, uiState, profile) → Folio` — pure, memoized, logic-free runtime around it. It deliberately deferred rendering (ladder's out-of-scope note, gap G-1). CB1 is the binding layer that makes the sealed function a *seated instrument*: the ComposerRuntime's home, the render of composed folios in Ledger material, and the seat-surface implementation that retires `ThrowawayFolio_DELETE_BY_DESIGN`. CB1 does NOT design folios (GENESIS 02 did), does not touch the composer's contracts, does not render the full `@ash-archive/ledger-ui` design-production layer (still the separate final campaign) — it builds the *minimum true bench*: the Codex stance's folios, composed live from a real vault, rendered plainly but lawfully.

## The three riskiest open questions
1. **What crosses the seat boundary?** `mount(host, ctx)` is frozen and deliberately silent about data. Does the shell hand the instrument a live core handle (vault session) through an extended `SeatContext`, or does the instrument import core itself and the shell pass only `worldId`? The first makes the shell a data broker (jurisdiction creep); the second makes every instrument open its own vault session (resume-budget risk, two sources of truth for "which world is open"). CB1 must pick, name it, and propose the `SeatContext` extension to the frozen contract *as an amendment, not a drive-by*. My lean: shell owns the vault session, passes a narrowed read/subscribe/append capability — but that capability's shape is exactly the risk.
2. **How much render is lawful before the design-production campaign?** The ladder explicitly defers exhaustive render specs (`@ash-archive/ledger-ui`) to a final campaign; CB1 needs *some* render now. The risk is CB1's "plain but lawful" bench quietly becoming the de-facto design system nobody sealed. Proposed line: CB1 renders sealed element *shapes* in raw Ledger tokens with zero minted visual identity beyond tokens, and every renderer carries a `// PROVISIONAL RENDER — design-production campaign supersedes` header the later campaign is licensed to delete wholesale. Is that line real or self-deception? The hostiles should attack it hard.
3. **Which core steps gate which CB1 stages?** Compose needs §5 (query/links — step 3, in flight). Inscribe/Strike need write paths (§3 — built). Bind needs §6 Binding (step 4, unbuilt) — but the Binding is the product's apex ceremony and SPEC-002's folios reference banked/full seals. Can CB1-α ship its first seated folio with Bind visibly absent (honest scaffolding, like the page-card), or does a bench without Bind violate "the six verbs are the grammar" (canon §4) badly enough that CB1-α must wait for core step 4? My lean: α seats read+inscribe truthfully and *names* the missing verbs as unbuilt rather than hiding them — burden of proof on ceremony, never on silence — but this is a canon-adjacent call and the prosecutor may convict it.
